An Introduction to Filters

Filters are mechanism to select a subset of buffers for display by bufdump. Filters and searches provide a powerful mechanism for looking for specific data within event buffers.

Filters operate on two levels:

  • Buffer type filtersspecify that you are only interested in seeing specific subsets of buffer types
  • Bit pattern filtersspecify that you are only interested in seeing buffers that have specific bit patterns or sequences of bit patterns in their bodies.

Filters are established with the Filter->Filter... command. If a filter has been defined, everytime you request the next buffer from a data source, bufdump will read and ignore buffers until the filter criterion is met. Note that at this time, there is no way to abort the next buffer operation. Therefore if you create filter criteria that are never met, bufdump will hang and need to be restarted. Later implementations of bufdump may lift this restriction.
